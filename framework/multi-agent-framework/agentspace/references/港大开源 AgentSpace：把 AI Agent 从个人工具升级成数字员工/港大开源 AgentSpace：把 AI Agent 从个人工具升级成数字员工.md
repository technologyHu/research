> 港大 HKUDS 最新开源的 AgentSpace，值得关注的地方不在于“又多了一个 Agent 框架”，而是它把 Agent 放进了团队协作场景：身份、任务、调度、权限、审批、审计都要被管理。对职场人来说，这可能是 AI Agent 从个人助手走向数字员工的关键一步。

你有没有发现，最近 AI Agent 越来越强，但真正放到团队里，反而变得很难用？

一个 Agent 能写代码，一个 Agent 能查资料，一个 Agent 能做表格，一个 Agent 能跑自动化流程。可问题是：它们都躲在各自的聊天框、终端、插件和账号里。

个人用起来很爽，团队一协作就乱了。

谁给 Agent 派任务？谁知道它做了什么？它能不能访问客户数据？执行前要不要审批？出错了谁来追溯？

这就是 AgentSpace 这类项目真正有意思的地方：它不是让某个 Agent 更聪明，而是开始思考一个更职场、更组织化的问题——**公司到底该怎么管理一群 AI Agent？**

## AgentSpace 是什么？

简单说，AgentSpace 是港大 HKUDS 开源的一个面向“人类 + Agent”的共享工作空间。

它不像传统 Agent 框架那样，只关心“怎么让一个 Agent 完成任务”。它更像一个团队操作台：人类、不同能力的 Agent、任务、权限、协作记录，都可以被放进同一个 workspace 里。

你可以把它理解成：

* Notion 是人的知识协作空间；
* Slack / 飞书是人的沟通协作空间；
* Jira 是人的项目协作空间；
* AgentSpace 想做的是：**人和 Agent 一起工作的协作空间**。

AgentSpace 官方 showcase 数字员工展板截图

这件事很关键。

因为过去我们聊 AI Agent，经常会陷入一个误区：只盯着“单个 Agent 有多强”。比如能不能自动写代码，能不能自动浏览网页，能不能自动生成报告。

但真正进入公司之后，强不强只是第一层问题。

更重要的是：它能不能被分配任务？能不能和其他 Agent 配合？能不能被限制权限？能不能留下过程记录？能不能被人类审核？

如果这些问题解决不了，Agent 就永远只是个人效率玩具，很难成为组织里的“数字员工”。

## 今天的 Agent 工作流，卡在哪里？

现在很多年轻职场人已经开始用 AI 提效了。

写周报用 ChatGPT，写代码用 Claude Code，整理资料用 Perplexity，做图用 Midjourney，自动化用 Dify、Coze 或 n8n。

单点效率确实提升了，但你把这些工具放进一个团队，就会遇到几个典型问题。

第一，**太私有**。

很多 Agent 都跑在个人账号、个人电脑、个人 API Key 里。你让它帮你做事没问题，但团队很难知道它到底做过什么。更麻烦的是，一旦人离职、电脑换了、账号停了，很多工作流就断了。

第二，**太分散**。

调研 Agent 在一个平台，代码 Agent 在另一个平台，数据 Agent 又在另一个平台。每个 Agent 都有自己的输入输出格式，团队成员之间只能靠复制粘贴来串联。

第三，**缺少协作层**。

一个复杂任务往往不是单 Agent 能解决的。比如做一份竞品分析，至少涉及资料搜集、数据清洗、内容总结、PPT 生成、负责人审核。现在很多流程仍然需要人类在中间反复转手。

第四，**缺治理**。

这是企业最在意的点。Agent 能访问哪些文件？能不能调用外部工具？能不能发邮件？能不能改数据库？哪些动作必须先让人类批准？

如果没有治理，Agent 越强，风险越大。

这也是为什么 AgentSpace 的方向值得看。它不是单纯追求“自动化更多”，而是试图给 Agent 加上一层组织管理能力。

## 四个核心能力：把 Agent 管起来

从官方展示和项目定位来看，AgentSpace 最值得关注的能力，可以概括成四个词：调度、共享、协作、治理。

先说调度。

一个团队里不可能只有一个 Agent。就像公司不会只有一个员工一样，不同 Agent 会有不同角色：有的擅长写代码，有的擅长查资料，有的擅长做数据，有的擅长生成文档。

这时候就需要一个类似 AgentRouter 的东西：根据任务类型，把任务分配给合适的 Agent。

比如你输入“帮我分析这个产品最近三个月的用户反馈，并生成一版改进建议”，系统可以拆成几步：

* 资料 Agent 收集反馈；
* 数据 Agent 做分类统计；
* 分析 Agent 提炼问题；
* 写作 Agent 输出报告；
* 人类负责人最后审核。

这比你在一个聊天框里来回追问，要更接近真实工作流。

AgentSpace 官方 showcase 中的 AgentRouter 界面截图

第二是能力共享。

以前我们每个人调教出来的 Prompt、工作流、工具调用链，往往只属于个人。一个同事做了一个很好用的“周报 Agent”，另一个同事未必能直接复用。

AgentSpace 这类 workspace 的价值在于：把 Agent 能力沉淀为团队资产。

一个内容团队可以有选题 Agent、资料 Agent、标题 Agent、初稿 Agent、校对 Agent、分发 Agent。一个研发团队可以有需求分析 Agent、代码 Agent、测试 Agent、文档 Agent。一个运营团队可以有活动策划 Agent、用户分析 Agent、数据看板 Agent。

当这些 Agent 不再藏在个人电脑里，而是进入团队空间，效率提升才有可持续性。

第三是协作。

多 Agent 协作不是简单地“让几个机器人一起聊天”。真正有用的协作，必须围绕任务、上下文和结果。

谁接了哪个任务？当前做到哪一步？中间产物在哪里？人类在哪个节点介入？失败后怎么重试？

这些都是工作流里的硬问题。

所以 AgentSpace 的多 Agent war room 概念很值得关注。它更像一个任务作战室：不同 Agent 不是随机发言，而是围绕同一个目标分工协作。

![实证：AgentSpace 官方 showcase 多 Agent war room 截图](实证：AgentSpace 官方 showcase 多 Agent war room 截图)

第四是安全治理。

这点可能没有“自动写代码”那么酷，但对组织来说最重要。

你不可能让一个 Agent 默认拥有所有权限。尤其是在企业环境里，权限、审批、审计都必须明确。

比如：

* 查公开网页，可以自动执行；
* 读取内部文档，需要授权；
* 发送外部邮件，需要人工确认；
* 修改线上数据库，必须禁止或走审批；
* 使用客户隐私数据，必须留下审计记录。

一个 Agent 如果想成为数字员工，就不能只会干活，还要可控、可查、可追责。

## 它不是普通 Agent 框架

很多人看到开源 Agent 项目，第一反应是：“这是不是又一个 LangChain？是不是又一个 AutoGPT？”

我觉得不太一样。

普通 Agent 框架更多解决的是“Agent 如何思考和调用工具”。比如任务拆解、工具调用、记忆、规划、执行循环。

AgentSpace 更像解决另一层问题：**Agent 如何进入组织协作**。

换句话说，它不只是技术框架，更像协作层、管理层、治理层。

如果用职场类比：

* 单个 Agent 像一个能力很强的实习生；
* Agent 框架像教它怎么做事的方法论；
* AgentSpace 像公司里的部门系统、权限系统、任务系统和汇报系统。

这也是未来 AI Agent 落地的关键分水岭。

个人使用阶段，我们追求的是“能不能帮我省时间”。团队使用阶段，我们追求的是“能不能稳定地、可控地、可复用地省时间”。

这两个问题完全不同。

## 和 Claude Code、OpenClaw、Codex 是什么关系？

最近一段时间，大家对 Claude Code、OpenAI Codex、OpenClaw 这类工具的关注度很高。

它们代表的是 Agent 在某些垂直场景里的能力突破，尤其是代码和自动化执行。

比如 Claude Code 很适合开发者在终端里让 AI 理解项目、修改代码、跑测试。Codex 更偏向把编程任务变成可委托的执行流程。OpenClaw 这类项目则探索更开放的 Agent 操作和任务执行方式。

但这些工具大多还是围绕“个人使用”或“单任务执行”。

AgentSpace 的关系更像是：它不一定取代这些 Agent，而是把它们纳入一个更大的协作空间。

你可以想象一个未来场景：

产品经理在 AgentSpace 里提交需求，需求分析 Agent 先拆解任务；研发相关部分交给 Claude Code 或 Codex 类 Agent；资料部分交给检索 Agent；测试部分交给测试 Agent；最后由负责人在工作台里审核结果。

也就是说，AgentSpace 可能不是“最强 Agent”，而是“管理所有 Agent 的空间”。

这对职场人的启发很直接：未来真正值钱的能力，可能不是“会用某一个 AI 工具”，而是会设计 AI 工作流，会管理 AI 协作，会把分散工具变成稳定产出。

## 内容团队可以怎么借鉴？

如果你是内容、运营、市场、知识管理相关岗位，AgentSpace 的思路特别值得抄作业。

不用等它完全成熟，你现在就可以按“数字员工团队”的方式改造自己的 AI 工作流。

比如一个内容团队，可以先搭一个最小可用的 AI 分工系统：

1. 选题 Agent：每天抓取行业热点，输出 10 个选题；
2. 调研 Agent：针对选题整理资料、链接和核心观点；
3. 结构 Agent：把资料变成大纲；
4. 写作 Agent：根据账号风格生成初稿；
5. 校对 Agent：检查事实、语气、错别字；
6. 分发 Agent：生成朋友圈文案、小红书标题、视频脚本。

这里重点不是“每个 Agent 多完美”，而是让流程固定下来。

你可以先用飞书文档、Notion、腾讯文档、Dify、Coze、n8n 组合出一个简化版。比如所有任务都进一个表格，每个 Agent 只处理自己那一列，最后由人类审核发布。

这其实就是 AgentSpace 背后的思想：把 Agent 当成团队成员，而不是临时工具。

![实证：AgentSpace 官方 showcase 多 Agent war room 截图](实证：AgentSpace 官方 showcase 多 Agent war room 截图)

对年轻职场人来说，这件事还有一个现实意义：你不一定要成为 AI 工程师，但你要学会成为“AI 协作经理”。

能把一个复杂任务拆给不同 AI 工具，能设计审核节点，能沉淀模板，能避免权限风险，这些都会变成新的职场竞争力。

## 国内 AI 产品会被怎么启发？

AgentSpace 也给国内 AI 产品一个很明显的信号：下一阶段的竞争，可能不只是模型能力，也不是单个聊天框体验，而是组织协作能力。

很多国内产品现在已经在做智能体、工作流、知识库、插件市场。但问题是，它们往往还停留在“创建一个智能体给个人用”的阶段。

真正进入企业之后，会需要更多基础设施：

* 统一的 Agent 身份管理；
* 团队级 Agent 权限配置；
* 多 Agent 任务编排；
* 人类审批节点；
* 全流程日志和审计；
* 可复用的团队 Agent 模板；
* 和飞书、钉钉、企业微信、GitHub、数据库等系统打通。

谁能把这些能力做扎实，谁就更有机会把 AI Agent 从“演示很酷”变成“每天都用”。

尤其对中小团队来说，这类工具的价值会非常直接。以前搭一个自动化系统要找开发，现在可能由业务负责人自己搭出一套“数字员工小队”。

这就是 AI 生产力真正普及的样子：不是每个人都去写代码，而是每个人都能组织一组 AI 帮自己完成工作。

## 你现在该怎么做？

如果你对 AgentSpace 感兴趣，建议不要只停留在“收藏 GitHub 项目”。

更实用的做法是三步。

第一，先去看官方 showcase，重点看它怎么展示 AgentRouter、数字员工展板、多 Agent war room 和权限治理。不要只看界面酷不酷，要看它解决了哪些协作问题。

第二，拿你自己的工作流做一次拆解。选一个重复度高、步骤清晰、结果可验收的任务，比如周报、竞品分析、会议纪要、客户调研、内容生产。

第三，把这个任务拆成 3-5 个“数字员工角色”。每个角色只负责一件事，并且规定输入、输出和审核标准。

比如“竞品分析”可以拆成：

* 信息收集员：只负责找资料；
* 数据整理员：只负责归类；
* 分析助理：只负责提炼观点；
* 文档编辑：只负责写成报告；
* 人类负责人：只负责判断和拍板。

你会发现，一旦这么拆，AI 工具就不再是一个万能聊天框，而是一套可以持续迭代的工作系统。

这才是 AgentSpace 这类项目最值得我们学习的地方。

## 写在最后

Agent 的下一站，不是更会聊天，而是更像一个可治理的团队成员。

它要有身份，有职责，有边界，有记录，有审批，也要能和其他 Agent 配合完成复杂任务。

对年轻职场人来说，真正的机会不是等一个完美 AI 工具出现，而是从今天开始，把自己的工作拆成可协作、可复用、可管理的流程。

未来会用 AI 的人很多，但能管理一支“AI 数字员工队伍”的人，才会真正拉开差距。你最想把哪一类工作交给数字员工？欢迎在评论区聊聊。
